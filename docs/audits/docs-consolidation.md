2. Built a fresh inventory of code reality (delegated to subagents): API routers + `include_router()` calls + per-router `APIRouter(prefix=…)` declarations in `api/main.py`, every Celery task + beat schedule entry in `scraper/sports_scraper/celery_app.py`, every pipeline stage file under `api/app/services/pipeline/stages/`, league config in `api/app/config_sports.py` and the scraper twin, settings fields in `api/app/config.py` and `scraper/sports_scraper/config.py`, services and profiles in `infra/docker-compose.yml`, CI workflows in `.github/workflows/`, the migration list in `api/alembic/versions/`.
3. Cross-checked each linked doc's claims against that inventory and against direct file reads. Recorded each discrepancy as Act / Justify (per the Actionability Contract).

## Intentional gaps

- **No re-write of `docs/api.md`'s endpoint catalog.** The file is 3,893 lines; it is a hand-maintained mirror of FastAPI's auto-generated OpenAPI schema. Per-endpoint verification of every documented route would multiply the line-by-line audit by an order of magnitude with no durable benefit — the next backend change will desync it again. Instead, this pass added the pointer to `GET /openapi.json` (the source of truth) and corrected the misleading single-base-URL claim. **Reason:** keeping the file in lock-step with code is not a docs problem, it is a tooling problem; the durable fix is to lean on FastAPI's schema and let the doc cover only the parts the schema cannot (auth flows, JSON wire conventions, rate-limit model). **What would bring full per-endpoint validation into scope:** an explicit ask to re-author `docs/api.md` as a thin auth/conventions guide and delete the per-endpoint duplication.
- **No deletion of `docs/changelog.md` (1,195 lines).** It is a release-narrative document, not a per-statement claim about current code; entries are dated and clearly historical. Spot-check passes (the most recent `2026-04-25` entry was verified against the `analytics_routes.py` `get_sport_teams` SSOT change and the `team_filters.py` module). **Reason:** historical changelogs are intentionally non-current; deleting them would lose institutional history. They are not subject to the "must reflect current reality" rule the same way reference docs are.
- **No edit to `docs/roadmap.md` phase status.** The roadmap describes intended (`⬜`/`🔄`) and completed (`✅`) work. The completed claims that I sampled (Phases 0–3) tie back to specific tables and endpoints that exist in code today. **Reason:** product status outside Phase 0–3 is a product-owner question, not a docs-validation one.
- **No archival of older audit reports.** The unlinked `*-report.md` files (security, ssot, error-handling) were *almost* deleted as orphans, but each is a code-grounded record of an applied hardening pass on the current branch. Linking them from the index (the action this pass took) earns their existence rather than orphaning or removing them.
